搭建github博客

创建github仓库

注意：在github不是所有的仓库都可以用来搭建github pages只有以下两种情况是可行的

1. 以username.github.io 为名字的仓库的master分支可以用来搭建仓库
2. 其他命名的仓库的gh-pages分支

具体的搭建方法可以参考这个连接

关于CNAME和A记录的解释可以参考这里

配置sshkey

如何配置sshkey

1.进入用户目录

cd ~

2.生成公钥和私钥

ssh-keygen -t rsa -C "邮件地址"#填你注册github的邮件地址就可以了

这样会生成两个文件一个是公钥文件，一个是密钥文件

3.github 上配置

登录GitHub系统；点击右上角账号头像的“▼”→Settings→SSH kyes→Add SSH key。

4.验证是否成功

$ ssh -T git@github.com # 注意邮箱地址不用改
第一次验证的时候会出现The authenticity of host 'github.com (192.30.252.129)' can't be established.RSA key fingerprint is 16:27:xx:xx:xx:xx:xx:4d:eb:df:a6:48.之类的东西这是正常现象，输入yes

验证成功会出现

Hi liuxianan! You’ve successfully authenticated, but GitHub does not provide shell access.的字样而此时你会看到在~/.ssh文件夹下多了一个known_hosts文件

此外还需要配置

git config --global user.name "usrname"
git config --global user.email "email"

hexo写博客

先安装node.js，npm是javascripts

包依赖的管理，npm已经j集成到node.js里面去了，所以
安装了node.js的同时就已经安装了npm这里是node.js的安装教程

建议以下操作还是在git.bash上来操作，git.bash可以使你在windows上使用和linux一样的命令行来操作，但是git.bash的安装版本不要太高，否则会有bug。我安装的最新版本的git.bashctrl +C就无法终止正在运行的进程。

安装hexo

npm install -g hexo

记得将路径加入到系统环境变量里面去否则每次敲命令都需要完整的路径

站点根目录初始化

在适合地方创建一个文件夹，进入然后执行 hexo init

你会在文件夹下看到如下的东西

接下来就介绍一下这些东西是什么，有什么作用

1.首先是node_modules文件夹：这个文件存放了你hexo的一些依赖包

2.scaffolds文件夹：这个文件夹存放了hexo new 命令的模板

layout 可以是

  • page 新建博客的菜单页面
  • post 新建博文
  • draft 没用过不知道

3.source文件夹：这个文件夹存放了博客页面生成的md文件

_posts文件夹是你写的博客的md文件夹，对于你写的每一篇博客里面都有一个blogtitle.md文件和与blogtitle同名的文件夹，这个文件是用来存放资源的，比如这篇博客的所使用的图片就全部存放在那里。

categories和tags文件夹是你的博客主页菜单栏的文件夹，初始的设置没有这两个菜单栏，需要你自己添加

4.themes文件夹：里面是存放的不同主题。

5._config.yml文件，这个文件是博客全局的配置文件，其他的文件目前还没有接触过不是很了解。

theme文件夹里面也有一个——config.ymld文件

题文件文件修改

如果对主题没有任何的修改，第一次启动的是landscape这个主题，具体的可以参考这里,这篇博客介绍的是next主题的修改

开始写自己的博客

在站点根目录下hexo new "博客名",然后编辑md文件就可以，这里使用在线的md文件编辑器

如果要在博客上添加图片参考这里.

将博客部署到github

上传到github

参考这里

记得把本机上的站点根目录备份到github上，这样就可以在任何主机上修改上传。还可以修改使手机访问。
